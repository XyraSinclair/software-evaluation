# Artifact judgment: snapshots A and B

## Scope and verification

I read the complete file inventory: 22 files in A and 18 in B. The snapshots have 18 paths in common; 12 are byte-identical, five substantive files differ (`README.md`, `install.sh`, and three skills), and the sixth difference is the worktree `.git` pointer. A alone has four `.forward/` records. Because shared files are byte-identical, a citation to one applies to both unless noted. Commit age was not treated as evidence of quality.

Checks performed:

- Ran each `install.sh` against a fresh `mktemp -d`: both exited 0 and installed all eight skill directories plus `forward/PRINCIPLES.md` (nine files). Repeated with a temporary `HOME` and `--user`; both again exited 0. With no argument, both exited 2 with usage text. Both scripts pass `sh -n` and are executable.
- `claude plugin validate A` and `claude plugin validate B` both pass under Claude Code 2.1.206. All 16 skill frontmatters parse as YAML; every `name` matches its directory and every `description` is nonempty.
- A's `hunt.jsonl` parses as JSON on all 10/10 lines; A's `worth.jsonl` parses on all 4/4 lines. B has no JSONL files. Semantic replay of A gives 10 probes, 5 finds, and 5 issue strings; worth totals are 10 probes spent and 25 worth-probes returned (ratio 2.5).
- The exact zero-event 95% Poisson upper bound is `-ln(0.05)/t = 2.995732274/t`, so the advertised rule-of-three approximation is sound. At the shared default `V=3`, however, stopping after only `t=3` gives upper-bound expected value `2.9957`, still almost three times the unit cost; `t=9` is the approximate break-even point (`0.9986`).
- All explicit local Markdown links resolve. Git provenance is readable: A is clean at `d2e5f03`, B is clean at `ecb4c6e`, and A's hunt records consistently name the latter as the pre-change probe head.

## Three worst defects in A

1. **Its load-bearing stopping rule contains an economic contradiction, and its self-audit missed it.** With its own default `V=3`, the low-stakes three-probe release is not licensed by the stated `3V` test; the upper-bound expected return is still about 3 per unit cost. Yet A logs `principles-math` as a zero-find probe and says the stopping math was clean ([A/.forward/hunt.jsonl:6](A/.forward/hunt.jsonl#L6), [A/.forward/cycle.md:17–20](A/.forward/cycle.md#L17-L20)). This is the worst kind of defect for this artifact: a denominator-driven audit falsely clears its own central denominator.

2. **The feedback calculation is not computable from the schema it ships.** A says to calculate per-area `V` from the worth returned by “the area's appraised changes” divided by finds those changes addressed ([A/skills/worth-ledger/SKILL.md:44–47](A/skills/worth-ledger/SKILL.md#L44-L47)), but a worth record has no area, hunt-issue ID, or `finds_addressed` field ([A/docs/ledger-schema.md:32–45](A/docs/ledger-schema.md#L32-L45)). Free-text change names and commit lists permit subjective reconstruction, not deterministic replay. The numeric `worth_probes` assignment is likewise unspecified.

3. **Its provenance is candid but stale and schema-loose.** The probe heads are genuine and the totals reconcile, yet every committed/appraised change still names its commit as `pending` ([A/.forward/worth.jsonl:1–4](A/.forward/worth.jsonl#L1-L4)), and two live records violate the documented lens vocabulary. There is no validation step to prevent either problem, even though the cycle calls these changes “shipped” ([A/.forward/cycle.md:32–37](A/.forward/cycle.md#L32-L37)).

## Three worst defects in B

1. **The central stopping rule is internally self-defeating.** The shared default says an issue is worth three probes, while the low-stakes branch permits stopping after three clean probes rather than nine ([B/skills/hazard-hunt/SKILL.md:29–40](B/skills/hazard-hunt/SKILL.md#L29-L40)). Its skill description's “provably low” language is therefore stronger than its own rule supports.

2. **Its calibration formula is not a formula.** `V = median(worth_probes) / find` has an undefined denominator, unclear aggregation, and no area linkage in the worth schema ([B/skills/worth-ledger/SKILL.md:42–46](B/skills/worth-ledger/SKILL.md#L42-L46), [B/docs/ledger-schema.md:32–45](B/docs/ledger-schema.md#L32-L45)). This breaks the promised feedback from appraisal into stopping thresholds.

3. **It asks users to trust an honesty system it has not run on itself.** There is no `B/.forward/` directory, so B supplies no self-denominator, provenance trail, calibration result, or truncation disclosure even though its benchmark contract says those must always accompany claims ([B/benchmarks/README.md:45–50](B/benchmarks/README.md#L45-L50)). Its first-run story is also misleading in plugin mode: `/forward` is not the namespaced plugin command, and the forward skill falsely says the root-level principles file “ships alongside” it ([B/README.md:55–63](B/README.md#L55-L63), [B/skills/forward/SKILL.md:8–11](B/skills/forward/SKILL.md#L8-L11)).

## Overall ordinal verdict

**A>B overall, with medium-high confidence (0.86).** A is not stronger because it is apparently later; it is stronger because the differing artifact repairs identifiable interfaces and conceptual seams, and—most importantly—contains an auditable self-run with denominators, provenance, provisional-appraiser labels, and explicit truncation. The lead is decisive on honesty infrastructure, clear on consistency and interface sharpness, and only narrow on correctness because both retain the same flaw in their central stopping rule and A's own audit incorrectly blesses it. Confidence is below “very high” because conceptual parsimony is partly qualitative and the actual model-trigger behavior was validated structurally, not by running an authenticated interactive Claude session end to end.
